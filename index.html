Jacob P Fodor | IST 411 Project Proposal

Realtime Collaborative Meme Studio

 

Project Description

I am proposing the development of a platform where anonymous users can come together to create memes, or other art from a blank canvas, collaboratively. I am proposing to facilitate this in a round-robin style, where users take turns making changes until the party of (2-4) individuals has agreed they are finished, and the product may be saved or added to a main gallery.

 

Project Design

Technologies

My intended foundation for this project is a server built with Spring MVC, in addition to a database be responsible for storing both finalized and raw images from a curated collection. These raw images will provide the initial “canvas” for users to play with and make changes to. The finalized images will have tags and other information regarding their creation. Due to the small size of most images used for creating memes, I believe storing them in SQL will be faster than leaving them in the filesystem.

Actual alterations to the canvas or images will require the use of a few libraries. Currently, I am examining using Java 2D graphics for making painted or shape changes to a canvas or image. This should be simple and freeform enough to implement. I also have some experience working with OpenCV, and intend to use it to apply filters and other effects to the collaborative products. Using this framework to create some presets for the users utilized through the UI should be both fast and effective.

For the interface of my application, I intend to use Bootstrap as the foundation. Bootstrap provides many components already stylized and very customizable. Using it should enable me to facilitate all of the interface elements I need, and produce a clean & modern interface.

As for the actual collaboration process, much of this will inevitably be worked out on my own. I don’t see a need to find software or some other workaround for setting up the creation session’s mechanics.

Use Cases

  • View created images in a gallery
    • Clicking on an image enlarges it, focuses, and presents to the user additional information such as creation date, tags, and download options.
  • Search for images
    • Search the entire gallery of images by their tags, or the date that they were created. This will not navigate the user from the main page, but will filter the images shown.
  • Creation session
    • Session begins 30 seconds after two users have queued for a session
    • Users can vote whether they would like a random image or a blank canvas, “coin flip” for a tie
    • Users will have ~15 seconds in a round-robin cycle to make a single alteration to the canvas
    • Session will end when:
      • 60 seconds after a single person indicates they are finished
      • Immediately following a turn where all users have indicated they are finished
      • One person remains in the session
  • Once completed, users will be prompted to add 1-2 tags (optional) to the image, giving an opportunity to add a signature or an identifying keyword if they’d like to find it in the gallery later

Checkpoints

Checkpoint #1

  • Server should be configured and setup
  • SQL DB architecture should be planned, and created
  • Gallery screen should be created with one or several “dummy” images proving it is interfaced with DB
  • Interfaces (button/field) for queuing up for a session and for searching gallery should be present but not functional

Checkpoint #2

  • Users should be able to queue for an anonymous creation session
  • Structure and turn system for the session should be implemented so that users can see an image, turns tick by, and a session can be completed (no editing)
  • User tagging system following session completion
  • Images should save to DB
  • Users return to gallery following a session ending
  • Frameworks for editing should be decided and be included in the project, though not necessary implemented yet

Checkpoint #3

  • Users should be able to search/filter the gallery by tags
  • During sessions, users should now have the ability to use at least:
    • One drawing tool
    • One text tool
    • Three filters
  • Image should be saved and refreshed for all users after each turn
  • System for voting on a blank canvas or random image implemented
  • DB populated with raw sample images

Justification

Novelty

Memes have a notorious reputation, and are sometimes as polarizing for people as politics. Regardless, their prevalence on today’s internet is indisputable. They are all over our Facebook and Instagram feeds, and indeed nearly every corner in between.

So often, however, these memes are created in a vacuum, by a single individual. For those who do enjoy memes, this is a satisfying and adequate way to consume them. I would like to argue that there is even more novelty to be had by stepping out of this isolation and passive process, by providing an anonymous opportunity for people to create something new together. By removing any noise or fluff and providing a streamlined creation process, my hope is that users will find ways of expressing themselves and perhaps even communicating in a fun and fast-paced environment.

Complexity

This project will be challenging because I am proposing to create a turn-based session environment that will have multiple users interfacing with the same canvas object. This will require me to build skills in structuring and regulating a session of this sort for multiple users. It will also require me to use some powerful frameworks, like Java 2D and OpenCV to do some customized and very specific tasks. Combining this with regulating the product throughout the session I would believe is an ambitious undertaking.

Additionally, I will have to support the session system with an adequate interface so that users can actually complete my use cases. This will require me to use an industry web tool like Bootstrap to create something far more complex than any templates likely available.

Finally, even if the context or “flavor” of memes is not your forte or interest, I believe this project has the potential to be enjoyed by a wide variety of users, while also demonstrating significant programming/technical proficiency.
